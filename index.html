Serverless Functions for Kubernetes

Fission is a serverless ("functions-as-a-service", a.k.a. Lambda) framework for Kubernetes.

With Fission, you write short lived functions and map them to HTTP requests (or other event triggers).

Deploy functions instantly with one command. There are no containers to build, and no Docker registries to manage.

$ cat hello.js
module.exports = function(context, callback) {
    callback(200, "Hello, world!\n");
}

$ fission function create --name hello --env nodejs --code hello.js

$ fission route add --function hello --url /hello

$ curl http://router.fission/hello
Hello, world!

Fast

Fission maintains a pool of containers so your functions have very low cold-start latencies, typically ~100msec.

Open

Avoid vendor lock-in: Fission is open source under the Apache License. It's also extensible to any language and environment.

Runs anywhere

Since it runs on Kubernetes, you can run Fission in your private datacenter, in a public cloud, or on your laptop.
